Q: Discuss the scope of var, let, and const

var:

  • Function-scoped or globally scoped.
  • Hoisted to the top of their scope.
  • Can be reassigned and redeclared.

let:
  • Block-scoped.
  • Hoisted to the top of their scope but not initialized until the declaration is encountered.
  • Can be reassigned but not redeclared.
const:
  • Block-scoped.
  • Hoisted to the top of their scope but not initialized until the declaration is encountered.
  • Cannot be reassigned or redeclared. The value assigned during declaration is constant.
Use const by default and only use let when you need to reassign a variable. Avoid using var in modern JavaScript.

Q: Tell us the use cases of null and undefined

Undefined: Represents the absence of a value or an uninitialized variable. Automatically assigned to variables that are declared but not assigned a value. Commonly used to check if a variable has been defined or initialized.
null:Represents the intentional absence of any object value. Typically set explicitly by the developer to signify the absence of a meaningful value. Often used in situations where an object or value is expected, but none is applicable or available.

Q: What do you mean by REST API?

REST API stands for Representational State Transfer Application Programming Interface. It is an architectural style for designing networked applications. REST is not a specific technology, but rather a set of principles and constraints for building web services.
